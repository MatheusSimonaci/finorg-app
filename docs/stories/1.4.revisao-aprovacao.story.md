## User Story

**Como** usuário,
**Quero** revisar as transações com classificação duvidosa e corrigir quando necessário,
**Para que** os dados no sistema sejam sempre precisos e reflitam a realidade.

---

## Acceptance Criteria

- [x] AC1: Tela lista todas as transações da importação mais recente (filtrável por batch)
- [x] AC2: Transações com `confidence < 0.75` são destacadas com badge "⚠️ Revisar"
- [x] AC3: Usuário pode editar `nature`, `category`, `subcategory` e `type` de qualquer transação via dropdown inline
- [x] AC4: Ao editar, campo de `confidence` é atualizado para `1.0` e `classificationOverride` para `true`
- [x] AC5: Botão "Aprovar tudo seguro" aprova em lote todas as transações com `confidence ≥ 0.75`
- [x] AC6: Toggle "Reembolsável pela empresa" disponível em transações `nature=empresa` ou `nature=work_tool`
- [x] AC7: Resumo no topo: "X importadas | Y aprovadas | Z aguardando revisão"
- [x] AC8: Botão "Confirmar importação" fica ativo apenas quando Z = 0
- [x] AC9: Filtro por: todas | aguardando revisão | aprovadas
- [x] AC10: Exibição do `reasoning` da IA disponível (segunda linha na célula de descrição)

## Dev Notes

### Lógica de estados da transação

```typescript
type ReviewStatus = 'pending' | 'approved' | 'overridden'

// pending: confidence < 0.75 e não foi revisada pelo usuário
// approved: confidence >= 0.75 (auto) ou usuário aprovou manualmente
// overridden: usuário editou a classificação
```

### Componente de linha da transação

```tsx
<TransactionRow
  transaction={tx}
  onCategoryChange={(cat) => updateClassification(tx.id, { category: cat })}
  onNatureChange={(nat) => updateClassification(tx.id, { nature: nat })}
  onReimbursableToggle={() => toggleReimbursable(tx.id)}
/>
```
