# STORY 7.2: Layout Mobile Responsivo do Snapshot

**ID:** 7.2 | **Epic:** [Epic 7 — Snapshot Mobile Estático](../prd/epic-7-snapshot-mobile.md)
**Points:** 3 | **Priority:** 🟠 High
**Created:** 2026-03-07
**Status:** ✅ Done
**Predecessor:** Story 7.1 (gerador de snapshot)

---

## User Story

**Como** usuário,
**Quero** que o snapshot tenha um layout otimizado para celular,
**Para que** eu possa visualizar meu resumo financeiro confortavelmente no smartphone.

---

## Acceptance Criteria

- [x] AC1: Layout responsivo a partir de 375px de largura (iPhone SE mínimo)
- [x] AC2: Navegação por 4 abas: **Resumo** | **Gastos** | **Portfólio** | **Sonhos**
- [x] AC3: Suporte a dark mode automático via `prefers-color-scheme`
- [x] AC4: Funciona offline (arquivo local aberto no browser sem conexão)
- [x] AC5: Fonte legível sem zoom: mínimo 14px para texto de dados
- [x] AC6: Charts adaptados para tela estreita (ex: legenda abaixo do gráfico, não ao lado)
- [x] AC7: Manifest básico para bookmark na home screen (`manifest.json` inline ou meta tags)

---

## Dev Notes

### Estrutura do snapshot mobile
```html
<!-- /snapshot/index.html — auto-contido -->
<html>
<head>
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <meta name="theme-color" content="#0f172a">
  <!-- CSS inline com media queries -->
  <style>/* todo o CSS aqui */</style>
</head>
<body>
  <!-- 4 abas com JS vanilla (sem framework externo) -->
  <!-- Dados em: <script>window.__DATA__ = {...}</script> -->
  <!-- Charts via SVG gerado no servidor (sem Recharts no snapshot) -->
</body>
</html>
```
